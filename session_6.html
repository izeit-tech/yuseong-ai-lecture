004단계 흐름

도구는 AI가 실행하고, 무엇을 할지 기준은 사람이 정합니다.

단계하는 일사람이 정할 것
① 진단데이터에 어떤 문제가 있는지 찾는다이게 진짜 문제인가
② 전처리문제를 정한 규칙대로 깨끗하게 정리한다어떤 기준으로 정리할까
③ 시각화피벗으로 집계하고 차트로 그린다무엇을 볼까
④ 대시보드차트를 한 화면에 모아 한눈에 본다누구에게 보일까
📌 더티 데이터 = 흙 묻은 채소 "관평" · "관평 동" · "관평동" — 사람 눈엔 같은 동, 컴퓨터엔 서로 다른 3개. 이대로 집계하면 한 동이 세 조각으로 흩어집니다. (실습 데이터: 민원 85건 · 12개 항목)
STEP 1

01진단 — 6가지 문제 유형

AI가 빠르게 찾아주지만, 진짜 문제인지 판단은 사람 몫입니다.

문제 유형대표 사례
표기 이상관평 / 관평 동 / 관평동
공백 섞임완료 / 완료␣␣ (끝에 빈칸)
형식 불일치2026/04/16 · 46139 · 04/11/26
결측치담당자 · 만족도 빈칸
중복같은 민원번호 두 번
이상치처리비용 9천만 원 (오입력)

① 기본형 — 파일 올리고 이렇게만

진단 · 기본형
이건 우리 구청 4월 민원 데이터야.
이 표에 데이터 품질 문제가 있는지 점검해줘.

어떤 항목에 어떤 문제가 있는지, 예시랑 건수까지 알려줘.

② 구조형 — 더 정확하게

진단 · 구조형
넌 데이터 분석 전문가야.
정리가 필요한 부분을 찾아줘.

아래 6가지로 확인해줘.
1) 표기 이상  2) 공백 섞임
3) 형식 불일치  4) 결측치
5) 중복  6) 이상치

'항목 / 예시 / 건수'를 표로 정리해줘.
STEP 2

02전처리 — 원본 보존이 핵심

규칙은 구체적으로 · 삭제보다 표시 · 원본은 절대 안 건드립니다.

전처리 (파일 올린 뒤 입력)
[가장 중요 — 원본 보존]
원본은 절대 수정하지 마.
정리 결과는 '정리본' 새 시트로 만들고 원본·정리본 둘 다 있게.

[정리 규칙]
· 동: 11개 표준 동명 통일('기준_동' 기준)
· 부서: '○○과'로 통일
· 처리상태/채널 통일
· 모든 공백 제거
· 접수일: YYYY-MM-DD
· 비용: 숫자만
· 빈 담당자: '미배정'
· 빈 값: '확인필요'
· 중복 민원번호: 최신 1건만
· 비용 5천만 원 초과: '검토필요' 표시 (삭제하지 말 것)

[출력] 원본·정리본 시트가 있는 엑셀 + 무엇을 어떻게 고쳤는지 요약표

진행상황 md 정리 (다음 단계로 넘기기)

AI가 만든 파일은 사라질 수 있으니 꼭 내 컴퓨터에 저장하고, 맥락은 md로 넘깁니다.

진행상황 md 정리
지금까지 진행한 작업을 다음 작업에 이어 쓸 수 있게
'진행상황 md'로 정리해줘.

· 데이터 개요 (무슨 데이터, 몇 건, 항목)
· 전처리 기준 (우리가 정한 규칙들)
· 완료한 단계 / 현재 단계 / 다음 단계
· 다음 작업에서 AI가 알아야 할 주의사항

대화 전체를 옮기지 말고, 다음 단계에 꼭 필요한
핵심 결정사항만. md 파일로.
STEP 3

03시각화 — 피벗 + 차트

차트 뒤에는 항상 '집계표(피벗)'가 있습니다. 비교는 막대그래프, 비율은 원그래프.

① 피벗 추천받기

피벗 추천
첨부한 민원 데이터(정리본)로 인사이트를 뽑으려고 해.
어떤 피벗테이블을 만들면 업무에 유용할지 5개 추천해줘.

각 추천마다: 무엇을 알 수 있는지 / 행·값에 넣을 항목 / 어울리는 차트 종류

추천 5선 (예시) — 동별 건수 · 유형별 건수 · 처리상태 비율 · 부서별 만족도 · 채널별 접수

② 피벗 + 차트 만들기

시각화 (피벗+차트)
동별 건수 · 유형별 건수 · 처리상태 비율 · 부서별 만족도 ·
채널별 민원건수 피벗테이블과 차트를 만들어줘.

[규칙]
· 정리본 데이터는 건드리지 말고, 목적별로 새 시트에 ('피벗_동별' 등 분리)
· 각 피벗 옆에 어울리는 차트도 같이
· 한글이 깨지지 않게 해줘
· '확인필요'·'검토필요' 표시된 값은 평균·집계에서 빼줘
· 차트마다 무슨 의미인지 한 줄 설명도

[출력] 원본·정리본·피벗 시트가 모두 있는 엑셀 파일
⚠ 품질 포인트 '한글 깨짐 방지' + '이상치 제외' 두 줄이 결과 품질을 좌우합니다.
STEP 4

04대시보드 — 한 화면에 모으기

차트 이름을 박지 말 것 — '앞에서 만든 차트'로 열어둬야 누구에게나 작동합니다.

① 엑셀 대시보드 시트

대시보드 · 엑셀 시트
지금까지 만든 피벗테이블과 차트를 모아 '대시보드' 시트를 만들어줘.

[구성]
· '대시보드' 새 시트에 앞에서 만든 차트들을 모아줘
· 맨 위에 핵심 숫자 요약 (총 건수, 완료율, 평균 만족도)
· 그 아래 앞서 제작한 차트 배치

[규칙] 다른 시트는 그대로 / 한글 깨짐 방지 / 위=요약 아래=상세 / 색상 2~3가지

② 웹페이지(HTML) 대시보드

대시보드 · 웹페이지(HTML)
지금까지 만든 피벗테이블과 차트를 모아 '대시보드' html를 만들어줘.

[구성]
· 맨 위에 핵심 숫자 요약 (총 건수, 완료율, 평균 만족도)
· 그 아래 앞서 제작한 차트 배치

[규칙] 다른 시트는 그대로 / 한글 깨짐 방지 / 위=요약 아래=상세 / 색상 2~3가지

05파일 다루기 규칙

올리기 1번 → 이어가기 → 받기 1번. AI 결과는 꼭 저장해 둡니다.

상황방법
진단 + 전처리파일 1회 올리고 같은 창에서 진행
피벗 → 차트 → 대시보드한 흐름이라 같은 창에서 이어가기
단계가 크게 바뀔 때새 창 + 정리본·md 다시 올리기
대화가 길어 AI가 헤맬 때새 창으로 전환
